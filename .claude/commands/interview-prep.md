# /interview-prep — 면접 대비 & 강점 노트 업데이트

프로젝트 작업 중 나온 기술 질문, 면접 포인트, 강점 발견을 개인 노트에 추가한다.

## 대상 문서

- `docs/interview-prep.md` — 기술 개념, 면접 Q&A, 프로젝트 설계 판단 이유
- `docs/my-strengths.md` — 핵심 강점, 성격·태도, 약점 대비

## 입력

$ARGUMENTS

- 입력이 비어있으면 → 최근 대화에서 면접/강점 관련 포인트를 자동 추출
- `강점:` 으로 시작하면 → `docs/my-strengths.md`에 추가
- `면접:` 또는 그 외 → `docs/interview-prep.md`에 추가
- 두 문서 모두 해당되면 각각에 추가

## 실행 규칙

1. 두 문서를 모두 읽는다
2. 입력된 내용 또는 최근 대화에서 포인트를 추출한다:
   - **면접 노트 대상**: 기술 개념 질문, 프로젝트 설계 판단 이유, 시니어가 물어볼 만한 포인트, 이력서에 적었지만 본인이 깊이 모르는 부분
   - **강점 노트 대상**: 새로 발견한 강점, 기존 강점의 추가 근거, 약점·대응 전략
3. 기존 문서에 중복되지 않는 새 항목만 추가한다
4. 면접 노트 항목 구조:
   ```
   ## 질문/주제

   **결론:** (한 줄 요약)

   - 상세 설명
   - 이 프로젝트에서의 적용 사례
   - **면접 답변 키워드:** (핵심 표현)
   ```
5. 강점 노트는 기존 섹션에 항목 추가하거나, 새 섹션을 만든다
6. 추가 완료 후 어떤 문서에 어떤 항목을 추가했는지 사용자에게 알린다

## 주의사항

- 두 파일 모두 .gitignore 대상 — 개인 학습용
- 솔직하게 적는다 (약점도 포함)
- "면접에서 이렇게 말하면 좋다"는 실전 답변 가이드 포함
