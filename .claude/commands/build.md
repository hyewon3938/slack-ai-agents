# /build — 기능 구현 (Sonnet)

`/design`에서 작성한 설계 파일을 읽고, 그대로 구현한다.
설계에 없는 변경은 하지 않는다.

## 입력

$ARGUMENTS

비어있으면 `.claude/plans/` 디렉토리에서 가장 최근 설계 파일을 자동으로 찾는다.
특정 설계 파일 경로를 지정할 수도 있다.

## 사전 로드 (자동)
- `CLAUDE.md` — 프로젝트 규칙
- `docs/conventions.md` — 코드 컨벤션

## 실행 단계

### 1단계: 설계 파일 로드
- `.claude/plans/` 디렉토리에서 최신 `.md` 파일 탐색
- 설계 파일을 읽고 전체 구현 계획 확인
- 이슈 번호, 브랜치명, 파일별 변경사항, 구현 순서 파악
- 현재 브랜치가 설계 파일의 브랜치와 일치하는지 확인. 다르면 체크아웃.

### 2단계: 구현

설계 파일의 **구현 순서 + 커밋 단위**를 그대로 따른다.

#### 구현 규칙
- **TDD 대상은 테스트 먼저 작성** → 실패 확인 → 구현 → 통과
- 설계의 Before/After 코드를 기반으로 정확하게 수정
- 신규 파일은 설계의 스켈레톤을 기반으로 작성
- DB 마이그레이션이 있으면 마이그레이션 파일 먼저 생성
- 커밋 시 이슈 번호 참조: `feat: 기능 설명 (#이슈번호)`
- **⛔ 커밋 전 보안 점검**: 시크릿, IP, 도메인 등 민감정보 포함 여부 확인

#### 설계와 다를 때
- 설계의 코드가 현재 코드와 맞지 않으면 (라인 변경, 함수 시그니처 다름 등):
  → 멈추고 사용자에게 "설계와 현재 코드가 다릅니다" 보고
  → 사용자가 판단: 직접 수정 or "Opus로 설계 업데이트 필요"
- **설계에 없는 변경은 절대 하지 않는다** (scope creep 방지)

### 3단계: 테스트

코드 구현 완료 후, 테스트 코드가 필요한지 판단한다:

- **작성 기준** (아래 중 하나 이상 해당하면 작성):
  - 순수 함수 / 유틸리티가 새로 추가됨
  - DB 쿼리 로직 / 데이터 변환이 복잡함
  - 엣지케이스가 많은 비즈니스 로직
  - 설계에서 TDD로 분류되었지만 아직 테스트가 없는 항목
- **스킵 기준** (아래에 해당하면 생략):
  - UI 컴포넌트만 변경 (시각적 확인으로 충분)
  - 단순 CRUD API 라우트 (보일러플레이트)
  - 설정/환경변수 변경만
- 작성 시: 테스트 실행 → 통과 확인 → 커밋
- 스킵 시: "테스트 불필요 — [사유]" 명시

설계 파일에 테스트 케이스가 명시되어 있으면 해당 케이스를 그대로 구현한다.

### 4단계: 코드 리뷰

구현 완료 후, `docs/conventions.md` 기준으로 자체 리뷰를 수행한다.

#### ⛔ 보안 감사 (최우선)
- 코드/커밋에 API 키, 비밀번호, 토큰, IP, 도메인 등 하드코딩 여부
- 새/변경된 API 라우트에 인증 검증 여부
- SQL 쿼리에 파라미터 바인딩 사용 여부
- 에러 응답에 내부 구현 정보 노출 여부

#### 코드 품질
- 버그 위험: 잠재적 버그, edge case 누락, 에러 핸들링 부족
- 성능: 불필요한 연산, N+1 쿼리
- 가독성: 네이밍, 함수 길이, 복잡도

#### 컨벤션 점검
- 네이밍 컨벤션 (파일: kebab-case, 변수: camelCase, 타입: PascalCase)
- import 정렬, 에러 핸들링 패턴
- `any` 사용 금지, named export 사용

#### 심각도 분류 + 처리
- 🔴 필수 수정 → 즉시 수정하고 커밋
- 🟡 권장 → 사용자에게 수정 여부 확인
- 🟢 참고 → 보고만

### 5단계: PR 생성
- `gh pr create`로 PR 생성
- 이슈 체크리스트 업데이트
- `docs/project-history.md` 업데이트 (결과, 배운 점)
- 설계 파일에 "구현 완료" 메모 추가

### 6단계: 결과 보고
- 구현 요약 (변경 파일 수, 커밋 수)
- 코드 리뷰 결과 요약
- PR 링크
- 테스트 결과

## 주의사항
- 이 스킬의 핵심은 **설계를 충실히 따르는 것**. 자의적 판단으로 설계를 벗어나지 않는다.
- 아키텍처 수준의 판단이 필요한 상황이 발생하면 즉시 멈추고 사용자에게 알린다.
- PR/커밋에 민감한 env 정보 포함 금지.
